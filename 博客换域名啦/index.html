还是简单点好

这两天一直在优化博客的页面啥的，然后就感觉现在这个博客名字不太好，作为一个博客也应该域名简单，这样也方便传播，而且我最开始搭建博客想要用的域名也是 left ，只是大部分都被别人买了。

今天思考了一番，还是应当简短域名，方便传播记忆（快一年了一个固定读者都没有o(╥﹏╥)o），说做就做。

left.pink

前面也说了，大部分好的都已经被注册，翻了好久才看到这个，相对可以也相对便宜，大概三十多软妹币，还是个折后价

更换域名

这也是我第一次更换域名，还是可以做个记录的；更换域名主要就是替换全站的链接，也就是替换数据库，一番搜索后我决定采用 wp-cli 更换。

安装工具
curl -O https://raw.githubusercontent.com/wp-cli/builds/gh-pages/phar/wp-cli.phar
chmod +x wp-cli.phar
sudo mv wp-cli.phar /usr/local/bin/wp

更换域名
cd /var/www/html/
sudo -u www-data wp search-replace ‘old.com’ ‘new.com’

申请证书

证书也需要重新申请；先把老的证书删除

rm -rf /etc/letsencrypt/live/www.example.com/
rm -rf /etc/letsencrypt/archive/www.example.com/
rm /etc/letsencrypt/renewal/www.example.com.conf

为了防止出错还可以吧 letsencrypt文件夹删除

然后申请

wget https://dl.eff.org/certbot-auto
chmod a+x certbot-auto
sudo ./certbot-auto –apache

这里我报错了，没有检测到80端口，这个改一下apache2站点设置就好了。

重定向

弄完这些还得解决重定向，毕竟我有那么几篇文章在谷歌搜索非常靠前，这可是辛辛苦苦积累出来的。

这里我采用的cloudflare，在page rule下简单配置下就好了
