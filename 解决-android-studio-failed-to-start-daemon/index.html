解决 Android Studio “failed to start daemon”

2019-11-23 折腾

文章目录

本文最后更新于 <span id="expire-date"></span> 天前，文中部分描述可能已经过时。

不知道为什么最近老是出现这个问题，网上搜了一堆方法全都解决不了，最后自己误打误撞解决了。

我也不知道导致这个问题的原因是什么，网上的说法是端口被占用，抛去原因，按照我这几个步骤就应该能解决。

1，断开手机和电脑连接，然后随便创建一个虚拟机启动

启动完成后应该能看到日志窗口：

2，把虚拟机关掉，连接手机，这时就应该成功连上了，如果不行照着步骤多来几遍

我觉得原因应该是端口被占用了，当你启动虚拟机时，AS会强行把占用端口的软件停掉，这样关掉虚拟机后连接手机就没问题了。
