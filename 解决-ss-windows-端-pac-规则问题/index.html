解决 SS windows 端 PAC 规则问题

很久以前我的 SS 就是老版本不升级了，原因是某次更新后，我发现 PAC 规则会有问题，体现在只要是 IP 地址，都会走代理，常用的网站斗鱼也会走代理，在初次更新时没发现这个导致白白跑了许多流量。

并且，无论我怎么修改规则，它都不生效，把文件中的 douyu 字段删除确实可以解决，但是软件会自动恢复 pac ，非常蛋疼。

我搜索了很多次也没有找到解决办法，索性就回到老版本了。

直到最近，我才找到了解决办法。

解决办法

使用Windows自带的记事本打开 Shadowsocks.exe 所在目录下的 gui-config.json 文件，然后将文件中的 "geositePreferDirect": false, 修改为 "geositePreferDirect": true, 保存退出后删除同目录下的 pac.txt 文件，再重启 ss 即可

官方说明是这样的：

简单理解就是黑白名单模式。
